0) Instalacja i konfiguracja gita[edytuj]

  • Ściągamy gita z https://git-scm.com/
  • Większosć opcji konfigurayjnych można zostawić tak jak proponuje to instalator, potrzebny nam w każdym razie skrót na pulpicie
  • odpalamy skrót żeby zobaczyć jak małe okienko terminalu proponuje nam windows, ułatwi to ustalenie poprawnych wymiarów
  • klikamy na skrót prawym przycikiem myszy, otwieramy właściwości, znajdujemy zakładkę "Układ"
  • zmieniamy wymiary okna na takie które umożliwią wygodną pracę; wymiar nie jest podany w pikselach tylko w znakach alfanumerycznych. Ja np. mam 239 na 70


1) Inicjalizacja GITa lokalnie i wgranie plików na BB (na laptopie)[edytuj]

  • W folderze domowym (na ogół jest to C:\Users\nazwa_usera ) towrzymy pusty plik o nazwie _netrc (taka jest nazwa w Windowsie, jeśli ktoś ma linuksa to proszę o informację). Nie _netrc.txt tylko _netrc. Bez rozszerzenia. Do pliku wpisujemy poniższe:
machine bitbucket.org
login mój_login_na_bitbucket
password moje_hasło_na_bitbucket
  • git init w folderze z projektem
  • git add ścieżka_do_folderu,`git add ścieżka_do_innego_folderu_lub_pliku dodajemy pliki i foldery do śledzenia
  • git commit -m "Pierwszy commit" robimy pierwszy commit, opis może być rzecz jasna inny
  • Na bitbuckecie wybrać z górnego paska "Repositories" i "Create repository"
  • Bitbucket sam podpowie jak zacząć.

Wystarczy kliknąć na np. "I have an existing project" I dowiemy się że powinniśmy zrobić: 

cd /path/to/my/repo
git remote add origin tu_bitbucket_poda_link
git push -u origin --all


2) Inicjalizacja GITa i pobranie repo z BB[edytuj]

  • instalujemy gita, konfigurujemy gita, tworzymy _netrc
  • Otwieramy git basha
  • Przechodzimy do folderu w którym ma być folder z projektem (np. do "htdocs") folder z projektem ma jeszcze nie istnieć
  • Otwieramy repo na bitbuckecie
  • Wchodzimy w zakładkę "Overview"
  • Odnajdujemy link do klonowania; Wygląda podobnie do https://tsuma534@bitbucket.org/tsuma534/rolladen-panel.git (To ten sam link który bitbucket podrzucił nam na początku, gdy wysyłaliśmy pierwszy commit)
  • Kopiujemy link
  • W gitbashu wpisujemy git clone skopiowany_link_do_repo nazwa_folderu_z_projektem
  • cd nazwa_folderu_z_projektem


3) wgranie zmian na serwer (normalne git add, git commit, git push, tylko może z opcją zadbania o znaki końca lini)​[edytuj]

  • jeśli pracuje jedna osoba i nie używa różnych edytorów z róznymi ustawieniami to nie powinna mieć problemów z zakonczeniami linii. Nie polecam tez kodzenia na boku, bez kontroli wersji. Poza tym trudno tutaj doradzić na zaś coś innego niż przesiadka na linuksa
  • kodzimy kodzimy
  • git add ścieżka_do_pliku dodajemy zmiany z plików z których zmiany nam się podobają
  • git commit -m "opis zmian, czy coś takiego" commitujemy zmiany
  • git push wysyłamy zmiany na bitbucket


4) pobranie zmian z serwera i zmergowanie[edytuj]

  • Jeśli pracuje jedna osoba i wykorzystuje gita w taki sposób jak opisujesz, to nie będzie miała potrzeby merdżować
  • git pull powinno w tym układzie załatwić całą robotę
